开发者工具基础

  1. 打开方式
方式 操作
快捷键 F12 或 Ctrl + Shift + I
直接打开 Console Ctrl + Shift + J
元素选择器 Ctrl + Shift + C
右键审查 右键页面 → “检查”（Inspect）

CTF 中第一步永远是：按 F12。

  1. view-source:

(1)view-source: 是浏览器支持的一种 URI 方案（URI Scheme），用于直接查看网页的原始 HTML 源代码。

浏览器收到 HTML 后通常会进行渲染（把代码变成你看到的网页），而 view-source: 会让浏览器跳过渲染，直接把原始代码以文本形式展示出来。

(2)基本用法

在浏览器地址栏中输入：

view-source:https://www.example.com

浏览器就会显示该 URL 对应的原始 HTML 源代码，而不是渲染后的页面。

(3)它不是网络协议

view-source: 虽然长得像 http:、https:，但它不是网络传输协议。

对比 http: / https: view-source:
作用 告诉浏览器去哪个服务器获取资源 告诉浏览器以源码形式展示页面
是否传输数据 是，向服务器发送请求 否，只是浏览器端的显示方式
服务器是否感知 是 否，服务器不知道你用了 view-source
有 RFC 标准 有 没有，属于浏览器私有实现

简单来说，view-source: 是浏览器内部的一个显示开关，不是网络协议。

(4)view-source: 与 Ctrl+U 的关系

两者功能基本相同，都是查看网页原始 HTML 源码。

view-source: Ctrl+U
使用方式 地址栏输入 view-source:URL 快捷键 Ctrl+U
作用对象 可以查看任意 URL 只能查看当前页面
显示内容 相同，都是原始 HTML 相同，都是原始 HTML

在 CTF 做题中，两者效果一样，选哪个都行。

开发者工具(元素)

(1)Elements 面板简介

Elements 面板是浏览器开发者工具中最常用的面板之一，用于查看和实时编辑网页的 HTML 结构与 CSS 样式。

在 CTF Web 题中，Elements 面板是排查隐藏信息的第一站。

(2)方式

方式 操作
快捷键打开 DevTools F12
直接切换到 Elements Ctrl + Shift + C（元素选择模式）
右键审查元素 右键页面任意元素 → 检查（Inspect）

打开 DevTools 后，默认就会停留在 Elements 面板。

(3)界面组成

Elements 面板由三个主要区域组成：

左侧：HTML 树形结构

  • 显示当前页面完整的 DOM 树
  • 点击标签前的三角箭头可以展开/折叠子元素
  • 选中某个元素后，该元素在页面上会高亮显示

右侧上方：Styles（样式）

  • 显示选中元素当前应用的 CSS 样式
  • 可以直接修改、添加、禁用 CSS 属性

右侧下方：Computed（计算后样式）

  • 显示元素最终生效的计算后样式值
  • 包含继承自父元素的样式

(4)基本操作

  1. 搜索元素

在 Elements 面板中按 Ctrl + F，会出现搜索框：

  • 输入 HTML 标签名搜索：div、input、script
  • 输入属性搜索：type="hidden"、class="secret"
  • 输入文本内容搜索：flag{、password、secret
  • 输入 CSS 选择器搜索：#login、.admin-panel
  1. 展开和折叠
操作 方法
展开一个元素 点击标签前的 ▶ 三角箭头
折叠一个元素 点击标签前的 ▼ 三角箭头
展开当前元素及所有子元素 按住 Alt 键点击三角箭头
折叠当前元素及所有子元素 按住 Alt 键点击三角箭头
  1. 选中页面元素

方法一： 直接在 HTML 树中点击某个标签

方法二： 点击左上角的 元素选择按钮（箭头图标），或按 Ctrl + Shift + C，然后在页面上点击任意元素，DevTools 会自动定位到对应的 HTML 代码

(5)编辑 HTML

  1. 修改元素的属性

双击某个属性的值，即可进入编辑状态：

<!-- 原始代码 -->
<input type="hidden" name="role" value="user" disabled>

修改 value： 双击 value="user" 中的 user，改成 admin，按回车确认


删除属性： 双击 disabled，全部选中后删除，按回车确认


添加属性： 双击标签名或已有属性，在弹出的编辑区域中添加新属性
